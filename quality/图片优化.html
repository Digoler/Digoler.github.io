# 图片格式和应用场景一览

图片格式 特点 应用场景
JPG/JPEG 有损压缩，压缩率高，不支持透明 大部分不需要透明图片的场景
PNG8 无损压缩+256色+支持透明 大部分需要透明图片的场景
PNG24 无损压缩+2^24色+不支持透明 大部分需要透明图片的场景
PNG32 无损压缩+2^24色+支持透明 大部分需要透明图片的场景
webp 压缩程度更好，在iOS webview有兼容性问题 安卓全部
svg矢量图 代码内嵌，相对较小 图片样式相对简单的业务场景

# 响应式图片

响应式图片是指根据不同设备屏幕大小和分辨率，提供不同尺寸和分辨率的图片，以达到更好的用户体验和页面性能。

下面是使用响应式图片的基本步骤：

  1. 准备不同尺寸和分辨率的图片。可以使用图片处理工具，如 Photoshop，Sketch，GIMP 等，根据不同设备的需求，生成不同大小和分辨率的图片。通常需要提供多种尺寸和分辨率的图片，以便在不同设备上都能够正常显示。
  2. 使用 HTML 标签 img，并添加 srcset 属性。srcset 属性指定不同尺寸和分辨率的图片文件路径及其宽度描述符，以便浏览器选择最合适的图片进行加载。例如：
<img src="image-200.jpg"
     srcset="image-200.jpg 200w,
             image-400.jpg 400w,
             image-600.jpg 600w,
             image-800.jpg 800w"
     alt="Responsive Image Example">

上面的示例中，浏览器会根据设备的屏幕宽度选择最合适的图片进行加载。例如，在宽度为 500 像素的设备上，浏览器会选择加载 image-600.jpg 文件。

<img src="image-200.jpg"
     srcset="image-200.jpg 200w,
             image-400.jpg 400w,
             image-600.jpg 600w,
             image-800.jpg 800w"
     sizes="(max-width: 600px) 100vw,
            50vw"
     alt="Responsive Image Example">

上面的示例中，如果屏幕宽度小于等于 600 像素，则图片宽度为 100%（即 100vw），否则图片宽度为屏幕宽度的一半（即 50vw）。

# 渐进式图片

渐进式图片（Progressive Images）是一种逐步加载的图片格式，它可以在网页加载时逐渐显示出来，从而提高用户体验和页面性能。

与普通的图片格式不同，渐进式图片是逐步加载的。它会先加载一张低分辨率的模糊图像，然后逐步加载更高分辨率和更清晰的图像。【注意与图片懒加载的区别】这个过程可以使用户感觉到图片逐渐变清晰，同时也可以提高页面加载速度，减少页面卡顿和等待时间。
