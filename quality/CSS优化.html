# CSS优化

# 提升CSS渲染性能

  • 谨慎使用expensive属性，如:nth-child伪类；position:fixed定位
  • 尽量减少样式层级数
  • 尽量避免使用占用过多的CPU和内存的属性
  • 尽量避免使用耗电量大的属性，如CSS3 3D transform、opacity等

# 合适使用CSS选择器

  • 尽量避免使用CSS表达式，如background-color:expression((new Date().getHours()%2)?"#FFF":"#000")
  • 尽量避免使用通配选择器
  • 尽量避免类正则的属性选择器

# 提升CSS文件的加载性能

  • 压缩和合并 CSS 文件：可以使用 CSS 压缩工具来缩小 CSS 文件的大小，同时将多个 CSS 文件合并成一个文件来减少 HTTP 请求的次数。这可以大大提高 CSS 文件的加载速度。
  • 使用 CDN：使用内容分发网络（CDN）来托管您的 CSS 文件可以减少加载时间。CDN 通常将文件存储在多个地理位置的服务器上，这样用户可以从最接近他们的服务器下载文件，减少了跨地理位置的延迟。
  • 避免使用 @import：避免使用 @import 导入其他 CSS 文件，因为它会增加额外的 HTTP 请求，导致页面加载时间延长。
  • 使用缓存：启用浏览器缓存可以使用户在多次访问您的网站时更快地加载 CSS 文件。可以通过设置适当的 HTTP 响应头来启用浏览器缓存。
  • 使用媒体查询：使用媒体查询来限制 CSS 样式的适用范围，只在特定的设备或屏幕大小下应用 CSS 样式。这可以减少下载不必要的 CSS 样式表，并加快页面加载时间。

# 精简CSS代码

  • 删除冗余 CSS：删除不需要的或未使用的 CSS 样式，以减小 CSS 文件的大小。可以使用浏览器的开发者工具来查找和删除未使用的 CSS 样式。
  • 简化选择器：使用更简单的选择器可以减小 CSS 文件的大小并提高页面的加载速度。避免使用嵌套选择器和过于复杂的选择器。
  • 使用 CSS 预处理器：CSS 预处理器可以使用变量和函数来生成 CSS 样式。这可以使 CSS 文件更简洁、易于维护并减小文件大小。
  • 使用 CSS 模块化：使用模块化的 CSS 可以使样式更易于管理和维护。模块化的 CSS 文件可以根据需要加载，以减小页面的加载时间。
  • 压缩和合并 CSS 文件
  • 使用缩写语句、删除不必要的0、删除不必要的单位（如px）等等

# 合理使用Web Fonts

  • 将字体部署到CDN上
  • 将字体以base64形式保存在CSS中并通过localStorage进行缓存

# CSS动画优化

  • 尽量避免同时动画
  • 优化动画的性能：使用 transform 和 opacity 属性来创建动画效果，可以获得更好的性能表现。transform 属性使用 GPU 加速，而 opacity 属性可以触发浏览器的合成层，从而减少重排和重绘。
  • 使用硬件加速：使用 CSS 属性 will-change 可以告诉浏览器哪些元素将会被动画处理，并在它们开始动画前提前将它们绘制在合成层中，从而提高动画的性能。
  • 避免在动画中使用 display:none：在动画中频繁使用 display:none 属性会导致浏览器频繁重排，从而降低动画的性能【Tip：我们可以将动画的position属性设置为absolute或者fixed，将动画脱离文档流，这样他的回流就不会影响到页面了】。
  • 减少动画中的帧数：在创建动画时，应该尽量减少动画中的帧数，从而减少浏览器的负担。通常来说，30 帧每秒已经足够流畅。
  • 避免使用 !important：在 CSS 中频繁使用 !important 属性会导致浏览器强制渲染，从而影响动画性能。
  • 延迟动画初始化
  • 结合SVG
